## Module/Layer Structure

- **Collections** (`src/collections/`): Payload CMS collection configs defining data schemas (Users, Media, Notes as prototype for Lessons)
- **App Router** (`src/app/`): Next.js App Router with `(frontend)` and `(payload)` route groups
- **Auth** (`src/auth/`): JWT-based authentication with role guards (student, instructor, admin)
- **Middleware** (`src/middleware/`): Rate limiting, role-based access control
- **Services** (`src/services/`): Business logic layer for CMS operations
- **API** (`src/api/`): REST endpoints auto-generated by Payload at `/api/<collection>`
- **Hooks** (`src/hooks/`): Custom React hooks for data fetching

## Data Flow

1. Client → Next.js App Router (Server Components)
2. Server Components → Payload Local API or REST API (`/api/<collection>`)
3. Payload → PostgreSQL via @payloadcms/db-postgres
4. Auth: JWT tokens include user roles for fast access checks

## Infrastructure

- **Docker**: docker-compose.yml with payload + postgres services; Dockerfile for standalone Next.js deployment
- **CI**: `pnpm ci` runs `payload migrate` then `pnpm build`
- **Environment**: .env.example with DATABASE_URL and PAYLOAD_SECRET
